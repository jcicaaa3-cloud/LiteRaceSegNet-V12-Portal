# Architecture and evaluation notes

## Model positioning

LiteRaceSegNet is positioned as a lightweight road-damage segmentation architecture. The design target is a binary or small-class road-damage mask where the positive class can be small, irregular, low-contrast, and boundary-sensitive.

## Architecture hypothesis

The model implements three testable assumptions:

1. **High-resolution local detail is useful for small damage boundaries.**
   The H/2 detail branch keeps local structure available until fusion.

2. **Low-cost context is still necessary.**
   The H/8 context branch and LiteASPP are intended to provide road-surface context without introducing a heavy decoder.

3. **Boundary signal belongs inside the fusion discussion.**
   The boundary prediction head produces a boundary logit that can supervise boundary learning and support feature modulation before final mask prediction.

## Public diagram wording

The GitHub diagrams describe the backbone as `custom LiteIR-style lightweight CNN`. MobileNetV2/V3-style efficient CNN principles are part of the design background, while the implementation path used for LiteRaceSegNet is `seg/core/lightweight_race.py`.

## Evaluation policy

The repository separates three types of evidence:

| Evidence type | Meaning |
|---|---|
| Reference evidence | Preliminary values already available for the lightweight model. |
| Pending final evidence | Baseline, Boundary IoU, latency, stress, and ablation values generated by official scripts. |
| Claim gate | A decision layer that determines which statements are supported after metrics are measured. |

## Device interpretation rule

CPU and GPU results remain separate tracks. CPU results answer field/no-GPU feasibility, while GPU results answer acceleration, throughput, and memory behavior.

## Current claim scope

The implemented model can pass smoke checks, the reference LiteRaceSegNet parameter count is 124,509 trainable parameters, and existing reference evidence shows reasonable small-validation masks. Baseline, latency, Boundary IoU, stress, and ablation values are handled by the final evidence scripts.

## Baseline and release layout

LiteRaceSegNet uses the custom LiteIR-style lightweight CNN path in `seg/core/lightweight_race.py`. SegFormer-B3 is the separate Transformer baseline path under `seg/transformer_b3/` and `02_SETUP_SEGFORMER_B3_HF.bat`.

The public package contains static pages, source files, visual assets, templates, and experiment summaries. External pretrained weights, fine-tuned checkpoints, Hugging Face caches, private datasets, and credentials stay outside the zip.

Short description:

> LiteRaceSegNet is a custom LiteIR-style lightweight CNN segmentation model. SegFormer-B3 is the separate Transformer baseline used for comparison.
