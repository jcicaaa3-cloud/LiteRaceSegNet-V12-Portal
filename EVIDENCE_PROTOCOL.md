# EVIDENCE_PROTOCOL — LiteRaceSegNet-V13

This document describes how the V13 evidence package records measured results for the public portfolio.

> **Reference only / research portfolio**  
> This document supports portfolio review. Values and visual examples are paired with their recorded experiment conditions and file locations.

## Categories

1. **Measured** — generated by a recorded script or reproducible check.
2. **Preliminary reference** — useful for explanation, while still separate from final evidence.
3. **Additional validation item** — a pending item that needs a recorded measurement.
4. **Excluded** — private data, raw masks, checkpoints, credentials, and other files kept outside the public portfolio.

## CPU / GPU separation

CPU and GPU results are reported as separate tracks.

- CPU: lightweight deployment feasibility, single-device constraints, low-resource inference.
- GPU: accelerated experimentation, batch processing, throughput-oriented validation.

A CPU/GPU comparison is meaningful only when the benchmark layout, device, precision mode, checkpoint condition, image size, batch size, and evaluation script are recorded together.

## Required fields for future metrics

- Dataset split
- Input resolution
- Batch size
- Device name
- Precision mode
- Script or notebook path
- Date of run
- Random seed if applicable
- Whether the value is measured, preliminary, or an additional validation item

## Claim level

Portfolio text stays close to recorded evidence. Statements about production readiness, SOTA ranking, or broad deployment belong to a later evaluation package with matching datasets, protocols, and artifacts.

## Baseline separation

LiteRaceSegNet evidence separates the proposed model path from optional comparison baselines.

- Proposed model wording: custom LiteIR-style lightweight CNN segmentation model.
- SegFormer-B3 wording: external comparison baseline/reference.
- Public evidence scope: SegFormer-B3 weights, fine-tuned checkpoints, Hugging Face cache files, private datasets, and credentials stay outside the package.
- Optional setup/training artifacts are handled after license and redistribution terms are reviewed.

## Architecture and license notes

LiteRaceSegNet is implemented as a custom LiteIR-style lightweight CNN segmentation model. The LiteIRBlock follows lightweight inverted-residual design ideas inspired by MobileNetV2/V3-style blocks while keeping the implementation custom.

SegFormer-B3 appears in `seg/transformer_b3/` and related scripts as an external comparison baseline. The proposed model path remains `lite_race`.

NVLabs SegFormer reference materials are treated as research-evaluation references; TorchVision is treated as BSD-3-Clause and Hugging Face Transformers as Apache 2.0.
